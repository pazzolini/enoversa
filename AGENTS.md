# Enoversa agent instructions

These instructions apply to the entire repository.

1. Read `docs/PROJECT_GUIDE.md` before making changes. It is the repository's central source of truth.
2. Follow the current user request when it intentionally overrides the guide, then update the guide if the decision is permanent.
3. Preserve the editorial voice. Do not invent wine facts, scores, prices, quotes, coordinates, visit details or technical production information.
4. Keep Selections, Portraits, Essays and Addresses visually and editorially distinct.
5. Validate frontmatter against `src/content.config.ts` and run proportionate checks after changes.
6. Preserve unrelated work in a dirty worktree.
7. Edit social templates in `scripts/render-instagram-assets.mjs`; generated PNG files live in `social/instagram/` and are not website assets.
8. Do not commit, push, deploy, alter Hostinger settings or change external services without explicit authorisation.
9. Never overwrite or deploy into the `legacy/` archive.
